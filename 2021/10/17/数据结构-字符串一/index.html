字符串

​ 无论是从抽象数据类型还是从具体实现的角度来看，字符串相对于之前所学的数据结构来说都更为简单。所以这里主要以讨论字符串匹配算法为主。

字符串的 ADT

  1. 定义：由来自字母表∑的字符所组成的有限序列:

    S=a0a1a2⋯an−1∈∑∗S = a_0 \quad a_1 \quad a_2 \quad \cdots \quad a_{n-1} \quad \in \sum_{}^* S=a0​a1​a2​⋯an−1​∈∑∗​

    这里所讨论的字符串，其长度都要远远大于字母表的长度。

  2. 相关术语与记法

    一般的,如果一个名为SSS的字符串，由nnn个字符构成，我们就将所有的字符从前至后，编号为000至n−1n-1n−1,并按照惯例记作S[0,n)S[0, n)S[0,n)。而串中秩为kkk的字符，也相应地记作S[k]S[k]S[k]。

    • 相等：s[0,n)=T[0,m)s[0, n)=T[0, m)s[0,n)=T[0,m)

      • 长度相等(n==m)(n==m)(n==m),且对应字符均相同(s[i]==T[i])(s[i]==T[i])(s[i]==T[i])。

    • 子串：S.substr(i,k)=S[i,i+k],0≤i<n,0≤kS.substr(i, k)=S[i,i+k],\quad 0 \leq i < n, 0 \leq kS.substr(i,k)=S[i,i+k],0≤i<n,0≤k, 亦即，从S[i]S[i]S[i]起的连续kkk个字符：

      \fbox{[0, i)} \quad \bbox[yellow]{\fbox{[i, i+k)}}\quad \fbox{[i+k, n)}

    • 前缀：S.prefix(k)=S.substr(0,k)=S[0,k),0≤k<nS.prefix(k)=S.substr(0,k)=S[0,k),0 \leq k < nS.prefix(k)=S.substr(0,k)=S[0,k),0≤k<n,亦即，SSS中最靠前的kkk个字符：

      \bbox[#9ff]{\fbox{[0,k)}}\quad\fbox{[k,n)}

    • 后缀：S.suffix(k)=S.substr(n−k,n),0≤k<nS.suffix(k)=S.substr(n-k,n),0\leq k < nS.suffix(k)=S.substr(n−k,n),0≤k<n , 亦即，SSS中最靠后的kkk个字符：

      \fbox{ [0,n-k)}\quad\bbox[#9ff]{\fbox{ [n-k,n)}}

    联系: S.substr(i,k)=S.prefix(i+k).suffix(k)S.substr(i,k)=S.prefix(i+k).suffix(k)S.substr(i,k)=S.prefix(i+k).suffix(k)

    • 空串：S[0,n=0)S[0,n=0)S[0,n=0),也是任何串的子串，前缀，后缀。
    • 长度严格小于原串的子串、前缀、后缀也称作真子串、真前缀与真后缀。

  3. 字符串的操作接口

    • length() : 字符串的长度。
    • charAt(i) : 对任意给定的秩也需要能直接返回对应的字符。
    • substr(i, k) : 子串
    • prefix(k) : 前缀
    • subfix(k) : 后缀
    • concat(T) : 将某个字符串T作为后缀，与当前字符串连起来。
    • equal(T) : 对于任意指定的字符串T，该接口可以判断T是否与当前字符串彼此相等。
    • indexOf(T) : 对于任意指定的一个长度为m的字符串P，该接口可以告诉我们，在当前的字符串中，是否存在某个子串与P完全相等。

蛮力匹配

​ 这类算法的思路与策略都是直截了当的，非常直观且易于理解。但是反过来，这个名字也暗示着它的效率是非常低下的。不过，这类算法也有它的存在价值，它可以帮助人们理解串匹配的计算过程，同时也为我们后续的改进提供了一个起点与参照。

​ 构思：自左向右，以字符为单位，依次移动模式串，直到某个位置，发现匹配。

​ 上面的蛮力匹配过程至少有两种实现的版本，因为由他们可以很方便的分别导出后续更为高效的算法。所以，这里对他们都进行介绍。

版本一：

​ 上面的算法中，jjj 的数值对应于在当前的对其位置下，已经做过的成功比对次数。因此，一旦jjj 达到了他的上界 mmm ,也就意味着模式串中的这 mmm 个字符，都得到了匹配。这是就是一次成功的匹配。此时，i−ji-ji−j 表示，在 i−ji-ji−j 处，发现了一次成功的匹配。

​ 当 iii 越界时，因为 iii 是逐一增加的，因此它越界时必然会恰好等于 nnn ,而此时的 jjj 依然处于合法的区间，即j<mj<mj<m, 综合这两个条件可以得出，此时退出时，返回值 i−ji-ji−j 必然会大于 n−mn-mn−m ,这时，n−mn-mn−m 应该是模式串相对于文本串而言，能够对齐的最靠右，也是最后一个位置，因此这时的 i−ji-ji−j 既然已经超越了这个合法的位置，这个算法的上层调用者就可以据此断定，整个匹配是以失败告终的。
